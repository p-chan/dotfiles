# AGENTS.md

## 共通

- 文字コードは UTF-8 を使用します

## コミュニケーション

- ユーザーとは、日本語で会話します
- ユーザーから意見を求められたときは、中立的・客観的に考えて意見します
- ユーザーに対して選択肢を提示して選んでもらうときは、何が推奨か明示したうえで、なぜ推奨なのか説明します

## 優先するコマンド

- `find` の代わりに `fd -u` を使います
- `grep` の代わりに `rg -uuu` を使います
- GitHub にアクセスするときは、`gh` を使います
- Web アプリケーションの UI の外観・動作を確認するときは、`playwright-cli` を使います
- 他リポジトリのコードを調べる場合は、`opensrc` を使います

## 依存関係[^1]

[^1]: ここで言う依存関係とは、外部のライブラリやフレームワーク、実行環境などのことを指します（例: npm パッケージや GitHub Actions など）

- 依存関係の情報（特にバージョンやオプションなど）は、記憶や推測だけで判断せず、必ず一次情報（コードやドキュメントなど）を確認します

## Git

- 各コミットはアトミックコミットを心がけます
- コミットメッセージでは「何をしたのか」と「なぜしたのか」を説明します

## GitHub

### Issue・PR の参照

Issue や PR を参照するときは、`#123` のような短縮形ではなく、フル URL の Markdown リンク形式で記述します。これにより OSC 8 対応ターミナルでクリック可能なリンクになります。

- Issue: `[#123](https://github.com/OWNER/REPO/issues/123)`
- PR: `[#123](https://github.com/OWNER/REPO/pull/123)`

リポジトリの URL は `gh repo view --json url -q .url` で取得できます。

| エイリアス                | 概要                             |

### `gh-notify-*` コマンドリファレンス

GitHub で非同期で実行される処理の終了を監視して通知するコマンドです。

| コマンド           | 概要                           |
| :----------------- | :----------------------------- |
| `gh-notify-checks` | チェックが終了したら通知する。 |
