# 既存プロジェクトへの導入

既存の tsconfig ファイルがある場合は `extends` だけを差し替え、他の設定は保持する。
存在しない場合は新規作成し、`include` は実際のソースディレクトリ構成に合わせる。

既存プロジェクトへの導入で型エラーが大量に出るなど、[strictest 系ベース](strictest-base.md)を重ねるのが適さない場合はユーザーに確認し、必要なら外す。

また、既存の `compilerOptions` に新しく重ねるベースと重複または矛盾する設定（例: `strict` を明示的に `false` にしている、緩い `target` を指定しているなど）が直接書かれている場合、ファイル直書きの設定は `extends` より優先されるため、型エラーを出さないまま strictest 系ベースが無効化されることがある。型エラーの有無だけで判断せず、該当する既存設定があれば残すかどうかをユーザーに確認する。
