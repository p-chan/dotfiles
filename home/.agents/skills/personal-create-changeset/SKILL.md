# changeset ファイル生成

## ルール

`changeset` コマンド（`npx changeset` など）は**使わない**。インタラクティブモードで起動するため、エージェントが操作を進められない。

代わりに、`.changeset/` ディレクトリに直接ファイルを作成する。

## フォーマット

ファイルパス: `.changeset/<ファイル名>.md`

ファイル名は `happy-bears` のような形容詞-名詞の組み合わせなど、他の changeset ファイルと被らない任意の名前にする。

```md
---
"<パッケージ名>": <major|minor|patch>
---

<説明>
```

### パッケージ名

プロジェクトの `package.json` を確認してパッケージ名を特定する。モノレポの場合は変更対象のパッケージを特定する。

### bump type

| 種別    | 目安                                   |
| ------- | -------------------------------------- |
| `major` | 破壊的変更                             |
| `minor` | 新機能の追加、後方互換性のある機能変更 |
| `patch` | バグ修正                               |

バージョンが `0.x.y` の場合、破壊的変更は `major` ではなく `minor` として扱う。

### 説明

- リリースノートに掲載することを意識する
- 変更の**主題**を 1 センテンスで書く
- 主題から逸れる補足情報は書かない（例: 主題が機能追加なら、付随するオプション名などは書かない。主題がオプション追加ならオプション名を書くのは当然 OK）
- **過去形にしない**（`Added` ではなく `Add`、「追加しました」ではなく「追加」「追加する」）
